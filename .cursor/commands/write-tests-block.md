# /write-tests-block

Используй для написания/обновления `pytest` тестов строго под заданный код-блок (функция/кусок логики), указанный в аргументах.

## Вход (как указывать блок)
- Укажи целевой блок в формате: `@path/to/file.py:START-END` (диапазон строк)
- Либо укажи файл целиком: `@path/to/file.py`

## Workflow
1. Test Writer анализирует указанный блок: какие функции публичные, какие инварианты/ошибки ожидаются.
2. Test Writer находит правильный тестовый модуль/папку в `tests/`:
   - если есть `src/`-layout — структура зеркалирует `src/`
   - иначе — структура зеркалирует импортный путь пакета (например, `demo_math/...`)
3. Добавляет минимальный набор тестов для:
   - happy path
   - edge cases
   - failure path (включая выбрасываемые исключения)
4. Test Runner запускает проверки в минимально достаточном объеме (предпочтительно точечно для модуля, относящегося к указанному блоку).
5. Если тесты падают — Debugger исправляет **только** причину провала в тестах или в затронутой бизнес-логике (без раздувания scope).

## Выход
- код тестов (или обновления)
- краткий report: что тестируется, какие кейсы покрыты, как проверено
