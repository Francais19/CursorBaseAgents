# /orchestrate

Используй для сложной Python-фичи или набора связанных изменений.

## Workflow
1. Planner создает план и task IDs.
2. Для каждой задачи:
   - Worker реализует
   - Test Runner проверяет
   - если тесты падают → Debugger исправляет
   - Reviewer смотрит diff и риски
   - при критичных находках → Debugger исправляет
3. Documenter готовит итоговый report.

## Правила
- не начинай реализацию без плана
- большие изменения режь на checkpoints
- лимит debug-итераций: 3
