# refactor-workflow

Используй для безопасного рефакторинга Python-кода без смены поведения.

## Перед началом
- убедись, что есть тесты или сначала инициируй их добавление
- зафиксируй цели рефакторинга: читаемость, разделение обязанностей, устранение duplication, снижение complexity

## Процедура
1. Опиши smells и ограничения.
2. Сохрани поведение и публичные контракты.
3. Делай рефакторинг поэтапно, малыми шагами (чекпоинтами).
4. Прогоняй tests/type-check после каждого существенного шага.
5. Составь отчет: что улучшено, что сознательно не трогалось.
