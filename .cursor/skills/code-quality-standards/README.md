# code-quality-standards

Стандарты качества кода для Python.

## Ожидания
- понятные имена
- маленькие функции с одной причиной изменения
- typing для публичных интерфейсов
- осмысленные исключения и сообщения об ошибках
- не дублировать бизнес-правила
- comments только там, где код сам не объясняет intent
- logs должны помогать разбору инцидентов
