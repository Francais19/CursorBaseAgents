# simple-workflow

Используй для `/implement` и простых задач в Python: функция, класс, endpoint, CLI-команда, background job, schema.

## Процедура
1. Уточни цель, контракт и место изменения по запросу пользователя.
2. Найди затронутые модули и существующие паттерны.
3. Реализуй код минимальным изменением.
4. Добавь или обнови тесты.
5. Запусти lint/type-check/test в разумном объеме.
6. Кратко задокументируй что изменилось и как проверить.

## Ограничения
- не делай архитектурный редизайн в простом workflow
- не меняй публичные контракты молча
- предпочитай маленький diff
