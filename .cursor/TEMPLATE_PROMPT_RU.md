# Prompt для актуализации шаблона `.cursor`

Ты получил шаблон `.cursor`, который нужно привязать к новому Python-проекту после переноса в другой репозиторий.

## Что нужно сделать
1. Сначала проанализируй новый проект и определи значения для плейсхолдеров `{{...}}`.
2. Подставь найденные значения в следующие файлы:
   - `.cursor/config.json`
   - `.cursor/rules/testing.mdc`
3. Создай `.cursor/mcp.json` в каталоге проекта для инструментов, специфичных для этого проекта, если такого файла еще нет.
4. Убедись, что `.cursor/config.json` и `.cursor/mcp.json` остались валидными JSON.
5. Остальные файлы `.cursor` не меняй.

## Как определять значения
- Используй код проекта, `pyproject.toml`, `requirements*.txt`, `poetry.lock`, `uv.lock`, `Pipfile`, `Dockerfile`, README, структуру каталогов и точки входа.
- Если значение можно надежно вывести из проекта, подставь его без запроса к пользователю.
- Если значение неоднозначно, сначала предложи наиболее вероятный вариант и коротко отметь допущение в итоговом отчете.
- Для `{{INTEGRATIONS}}` подставляй именно JSON-массив строк, а не строку с JSON внутри строки.

## Какие значения нужны
- `{{APP_FRAMEWORK}}` — строка, например: `FastAPI`, `Django`, `python-telegram-bot`, `aiohttp`, `none`
- `{{ASYNC_STACK}}` — строка, например: `true`, `false`, `mixed`
- `{{ORM}}` — строка, например: `none`, `sqlalchemy`, `django-orm`, `tortoise`
- `{{PACKAGE_MANAGER}}` — строка, например: `pip`, `poetry`, `uv`, `pipenv`
- `{{INTEGRATIONS}}` — JSON-массив строк, например: `["openai", "redis", "postgres"]`
- `{{ASYNC_HANDLERS_CONTEXT}}` — строка с кратким описанием, где именно используется async, например: `ASGI HTTP endpoints`, `Telegram bot handlers`, `background jobs`

## Формат работы
1. Определи значения плейсхолдеров по проекту.
2. Внеси правки в указанные файлы.
3. Верни краткий отчет:
   - какие плейсхолдеры были заменены
   - какие значения были определены автоматически
   - какие допущения были сделаны, если они были
   - что это улучшает для воркфлоу `.cursor` в новом проекте
